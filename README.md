# RAG-PROJECT
A RAG-based AI Knowledge Assistant that allows users to query their own documents and get accurate, context-grounded answers from a local LLM. The system ingests documents via Apache Tika, splits them into semantic chunks, and stores vector embeddings in PostgreSQL using pgvector. At query time, hybrid search — combining vector similarity and keyword matching — retrieves the most relevant chunks, which are injected into a carefully engineered prompt sent to an Ollama LLM. A two-layer Redis caching system handles both exact-match response caching and semantic caching using cosine similarity, reducing repeated LLM calls by up to 90% and achieving sub-5ms response times for cached queries. The entire system is built on a clean Spring Boot layered architecture — Controller, Service, Repository, and Config — ensuring maintainability and production readiness. Designed to eliminate hallucination, reduce compute cost, and serve accurate answers using only trusted internal data.
